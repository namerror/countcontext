ChatGPT Context Counter

How We Estimate Context Usage

This extension estimates how much of your ChatGPT context window is being used by the current conversation. The numbers are heuristic by default and can be customized in the panel.

How We Estimate Tokens

  • We parse visible conversation text and normalize whitespace.
  • Tokens are estimated from character counts (about 4 chars per token by default).
  • GPT-4 style models use a slightly tighter estimate (~3.8 chars per token).
  • GPT-3 style models use ~4 chars per token.
  • Attachments add a fixed estimate per detected attachment.

Overhead Assumptions

  • Base system/format overhead: 500 tokens.
  • Attachments: 200 tokens per detected attachment.
  • Token estimate: ~4 chars/token (GPT-4 uses ~3.8, GPT-3 uses ~4).

You can override plan, model, and context size directly in the overlay panel. This page reflects the default assumptions used before overrides.
